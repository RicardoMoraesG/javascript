Pushipin

O 'pushipin' posiciona o elemento de acordo com a rolagem da tela.

Possui 3 estados:

Um acima e abaixo do limite de rolagem e o estado fixado em que o elemento se torna fixo.

Como o pushipin altera o posicionamento, o elemento pode aparecer diferente quando os estados mudarem. A documentação do Materialize recomenda as classes abaixo para estilizar corretamente os 3 estados:

/* Classe para quando o elemento estiver acima do limite*/

.pin-top {

position: relative;

}

/* Classe para quando o elemento estiver abaixo do limite*/

.pin-bottom {

position: relative;

}

/* Classe para fixar*/

.pinned {

position: fixed !important;

}

Dropdown

A classe 'dropdown' pode ser usada para criar um menu suspenso, que só aparece se o usuário interagir com o ponto de ação.

Depende de codificação JavaScript, para os dropdowns da nav utilizei o seguinte código:

//começa o evento quando a página é carregada:

document.addEventListener('DOMContentLoaded', function () {

//variável que manipula a classe:

var elems = document.querySelectorAll('.dropdown-trigger');

//opções: aparecer abaixo do nome; aparecer ao passar o mouse:

var instances = M.Dropdown.init(elems, {

coverTrigger: false,

hover: true

});

});

Cards

A classe 'Card' formata a apresentação semelhante a um cartão.

Nesse caso, foi utilizada para conter as informações do conteúdo principal, entre imagens e textos.

A estrutura básica em CSS têm os seguintes atributos:

.card {

position: relative;

margin: 0.5rem 0 1rem 0;

background-color: #fff;

-webkit-transition: -webkit-box-shadow .25s;

-webkit-box-shadow .25s;

-transition: box-shadow .25s;

transition: box-shadow .25s, -webkit-box-shadow .25s;

border-radius: 2px;

}

Essas informações estão em um card

Sidenav

A classe Sidenav inclui uma barra de navagação lateral recolhível, interessante para utilização em telas pequenas por não concorrer espaço com o conteúdo enquanto o usuário não precisar dela.
